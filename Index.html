Polaris

Polaris Search is an application which integrate and facilitate all type of operation search. It purposes is to bring everything in a onetime search and provide operation information & useful link to external apps. It is built in a manner that individual user can customize search, views, links returned, create reports and promote them for other users to see them.

Polaris Features

Polaris is a set of Applications and reports developed on Polaris Application Platform utilizing Polaris Report Platform and Polaris Data Platform. Polaris enables development of new reports, new contents (such as blogs, discussion boards, training, working documents - example Dk/Df data, other engineering data), new analysis, or new applications by literally anyone. Polaris Data Platform enables access to data, information, and insights to Polaris Apps, Polaris Reports and Polaris Intelligent Product History.

Knowledge Based Solution (KBS)

Knowledge Based Solution (KBS) provides the search capabilities over text information like Failure Analysis (FA) cases notes, Services Request (SR) notes. KBS also support analytical capabilities to analyze FA data to identify root cause in FA process. Search for an FA case and see view of field history of Serial Number

Test Strategy Analytics (TSA)

The dashboard provides the starting point for the test strategy developer to analyze all test, repair, component, and measurement data for all products, all test areas, and all time frames. A set of commonly required analyses such as first pass yield, defect and symptom paretos, defective rate trends, and test time trends are provided.

Software Quality Dashboard (SQD)

Software Quality Metric Dashboard (SQD) provides consolidated view Level 1 metrics which include SWDPMH (Software Defects per Million Hours), URC (Universal Release Criteria) & iCFD (immediate Customer Found Defects). The capability provides current performance of these metrics quarter over quarter, month over month. Historically report metrics information is preserved for reference.

On Board Failure Log (OBFL)

Parser - Parse unstructured informaiton to tranform it into structured information leading. Extractor -Extract useful information from unstructured data and enable advanced decision making capability. e.g., read FA cases information from free text to identify specific cause of failure in the product.
